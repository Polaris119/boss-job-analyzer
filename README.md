# BOSS直聘岗位分析助手 MVP

一个零后端、用户主动触发的 Chrome Manifest V3 扩展。用户上传 PDF 简历并在独立页面校对，在岗位详情页点击悬浮按钮后，使用自备的 DeepSeek 或 OpenAI-compatible API 生成：

- 岗位画像、当前准备状态与简历差距诊断
- 投递行动和需要学习、复习或实践的知识点
- 短期/长期学习路线和个性化面试题
- 可编辑的定制简历
- 可选择仅生成岗位分析报告，跳过定制简历调用
- 简历区块/条目新增、删除和排序
- 自定义 PDF 简历主题色
- PDF 打印页
- Markdown 分析报告
- 岗位分析工作台和历史记录
- 1～3 个用户可配置的并行任务

## 安装

1. 打开 Chrome 的 `chrome://extensions/`。
2. 开启右上角的“开发者模式”。
3. 点击“加载已解压的扩展程序”。
4. 选择本项目目录。

项目不需要安装依赖或执行构建命令。源码已经按扩展入口、业务功能、浏览器适配和共享代码分层，选择项目根目录即可直接加载。

## 项目结构

```text
src/
├── entries/          # service worker、content script 和各扩展页面入口
├── features/         # 岗位采集、简历解析、AI 分析、任务队列和导出
├── platform/         # Chrome API 与 IndexedDB 适配层
└── shared/           # 常量、通用 UI、工具函数和设计变量
tests/                # 单元测试、架构契约测试和浏览器预览页
vendor/               # 随扩展发布的 PDF.js
assets/               # 扩展图标等静态资源
```

扩展页面和 service worker 使用原生 ES Modules。Manifest 按依赖顺序加载 BOSS content script 的多个经典脚本，因此在不增加构建步骤的情况下仍能保持模块边界。

## 验证流程

1. 打开插件侧边栏，上传一份文本型 PDF 简历。
2. 在自动打开的简历校对页修正分段和换行，然后点击“保存并完成同步”。
3. 配置 DeepSeek，或填写自定义 OpenAI-compatible Base URL、Model 和 API Key。
4. 点击“测试连接”。
5. 打开一个岗位详情页。
6. 点击右下角“AI 分析当前岗位”。
7. 在侧边栏选择是否生成定制简历，然后点击“加入分析队列”，扩展后台会开始执行，工作台可随时关闭。
8. 可以继续打开其他岗位，将多个任务加入队列；后台默认同时执行 2 个任务。
9. 任务完成后从工作台点击“查看结果”。
10. 在“岗位准备分析”标签通过左侧导航检查状态、差距、行动、学习路线和面试准备，并下载 Markdown。
11. 在“定制简历”标签编辑区块、条目和主题色，然后打开 PDF 导出页。
12. 在打印窗口中选择“另存为 PDF”。

生成岗位准备分析会依次调用岗位画像、差距诊断和准备规划三个 AI 节点；开启“生成定制简历”后会再调用一次。所有模型费用由用户自己的 AI 服务商收取。

## 数据与权限

- 插件只在 `www.zhipin.com` 上自动加载岗位页面识别代码。
- 完整职位只有在用户点击相应按钮后才会读取；PDF 只在用户主动上传后于本机解析。
- API Key 保存在 `chrome.storage.local` 或当前浏览器会话中，不经过本项目服务器。
- 自定义 AI 域名权限在用户保存配置时按域名申请。
- 基础简历和 AI 配置保存在扩展本地存储中；岗位任务、简历快照和分析历史保存在 IndexedDB。
- 插件不读取 Cookie，不调用 BOSS 未公开接口，不自动投递或发送消息。

## 当前限制

- BOSS 登录后的真实 DOM 仍需实机验证。页面选择器集中在 `src/features/job-capture/job-extractor.js`，PDF 文本分段逻辑在 `src/features/resume/resume-parser.mjs`。
- BOSS 若只通过动态私有字体展示薪资，插件会隐藏乱码；可读元数据存在时会自动恢复薪资。
- 当前只支持文本型 PDF；扫描件会提示暂不支持 OCR。
- 第一版通过浏览器打印窗口生成 PDF，不直接生成 DOCX。
- 自定义接口需要兼容 Chat Completions 响应结构：`choices[0].message.content`。
- 若接口不支持 `response_format`，插件会在 HTTP 400 后自动以普通 JSON 提示词重试。
- 没有跨设备同步；清除扩展数据会删除本地简历和历史。

## 岗位工作台

- 并行数可设置为 1～3，默认 2。
- 降低并行数不会中断已经运行的任务，只影响后续任务。
- 每个任务会冻结入队时的岗位、JD、基础简历和简历生成选项，避免切换岗位或开关后数据串联。
- BOSS 岗位 ID 用于判断岗位身份，标准化 JD 的 SHA-256 用于判断内容版本。
- 同一岗位、同一版本再次加入时，可以打开已有结果或使用当前简历重新分析。
- 已完成任务支持查看、重新分析和删除；失败任务支持重试。
- 工作台的“清空历史记录”只删除已完成、失败和取消的任务，不影响等待中或运行中的任务。
- 基础简历可在侧边栏上传按钮旁单独删除，删除前会要求确认。
- 岗位分析由扩展后台执行，关闭工作台不会中断任务；浏览器意外中止时会从最近已保存阶段恢复。
- 关闭整个 Chrome、电脑休眠或断网时无法继续请求，恢复浏览器与网络后会重新调度未完成任务。

## 常见问题

如果更新或重新加载扩展后出现“Extension context invalidated”，说明已经打开的 BOSS 标签页仍保留旧版脚本。刷新该 BOSS 页面后即可恢复。新版会在页面内显示中文刷新提示，不再弹出英文底层错误。
